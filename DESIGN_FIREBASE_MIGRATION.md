# Natick CNY 2026 - Firebase Migration Design Doc

**Status:** Draft
**Target:** Future Backend Integration
**Goal:** Migrate from `localStorage` to Google Firebase (Firestore) to enable multi-device concurrency, real-time updates for staff, and data persistence.

---

## 1. Architecture Changes

The current application is synchronous (reading directly from memory/local storage). Moving to Firebase requires shifting to an **Asynchronous** model.

### Key Shift: The Service Layer
We need to abstract `services/dataService.ts` into an interface to support swapping the "backend" easily.

**Current (Sync):**
```typescript
const reservations = getReservations(); // Returns array immediately
```

**Target (Async):**
```typescript
const reservations = await firebaseService.getReservations(); // Returns Promise
// OR
firebaseService.subscribeToReservations((data) => setReservations(data)); // Real-time listener
```

## 2. Firestore Schema Design

Firestore is a NoSQL document database. We will use a single top-level collection `reservations`.

### Collection: `reservations`
**Document ID:** Auto-generated by Firestore (e.g., `7f8a9s8d7f`).
**Index Fields:** `phoneNumber`, `id` (The "CNY26-XXXX" public ID), `contactName`.

#### Field Mapping

| TS Field (types.ts) | Firestore Field | Type | Notes |
| :--- | :--- | :--- | :--- |
| `id` | `publicId` | String | The "CNY26-XXXX" readable ID. Indexed. |
| `createdTime` | `createdAt` | Timestamp | Firestore Timestamp object. |
| `ticketType` | `ticketType` | String | 'EarlyBird' \| 'WalkIn' |
| `contactName` | `contactName` | String | |
| `phoneNumber` | `phoneNumber` | String | Digits only. Indexed for search. |
| `email` | `email` | String | Optional |
| `adultsCount` | `adultsCount` | Number | |
| `childrenCount` | `childrenCount` | Number | |
| `totalPeople` | `totalPeople` | Number | |
| `totalAmount` | `totalAmount` | Number | |
| `paidAmount` | `paidAmount` | Number | |
| `paymentStatus` | `paymentStatus` | String | 'Unpaid' \| 'Paid' |
| `checkInStatus` | `checkInStatus` | String | 'NotArrived' \| 'Arrived' |
| `lotteryNumbers` | `lotteryNumbers` | Array<String> | e.g. `["101", "102"]` |

### Collection: `system` (Singleton)
**Document ID:** `config`
Used to manage global counters or toggle registration status.

```json
{
  "isRegistrationOpen": true,
  "earlyBirdDeadline": "2026-02-15T23:59:59Z",
  "lotteryCounter": 108  // Used to generate sequential lottery numbers atomically
}
```

## 3. Implementation Steps

### Phase 1: Environment Setup
1.  Create project at [console.firebase.google.com](https://console.firebase.google.com).
2.  Enable **Firestore Database**.
3.  Enable **Authentication** (Email/Password for Staff).
4.  Run `npm install firebase`.
5.  Create `src/firebaseConfig.ts` to initialize the app.

### Phase 2: Refactoring `types.ts` & Services
1.  **Update Types:** Ensure types accommodate Firestore Timestamps (which differ from JS `Date.now()`).
2.  **Create Interface:**
    ```typescript
    export interface IDataService {
      createReservation(data: Partial<Reservation>): Promise<Reservation>;
      getReservation(publicId: string): Promise<Reservation | null>;
      updateReservation(id: string, data: Partial<Reservation>): Promise<void>;
      subscribeToAll(callback: (data: Reservation[]) => void): Unsubscribe;
    }
    ```
3.  **Update UI Components:** Refactor `PublicRegistration.tsx` and `StaffPortal.tsx` to handle `loading` states and `await` calls.

### Phase 3: Validation Logic (Rules)
Create `firestore.rules` to secure data.

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Public Registration (Create only)
    match /reservations/{docId} {
      allow create: if true; 
      
      // Public Read: Only if they know the Phone + Name (Implemented via Edge Function or simple Client Query)
      // For simplicity in Client-Side App:
      allow read: if true; // ⚠️ Warning: exposes data. Better to use Query constraints.
      
      // Staff/Admin Full Access
      allow write, update, delete: if request.auth != null && request.auth.token.role == 'staff';
    }
  }
}
```

### Phase 4: Atomic Counters (Lottery)
For the "Lottery Number" generation on Check-in, use a **Firestore Transaction**.
*   Current app uses `Math.random()`.
*   Firebase app should Increment a global counter in `system/config` to ensure no duplicate tickets are issued (e.g., Ticket #100, #101).

## 4. Steps for Connection

1.  **Install SDK:**
    ```bash
    npm install firebase
    ```
2.  **Initialize:**
    ```typescript
    // services/firebase.ts
    import { initializeApp } from "firebase/app";
    import { getFirestore } from "firebase/firestore";

    const firebaseConfig = {
      apiKey: "API_KEY_HERE",
      authDomain: "natick-cny-2026.firebaseapp.com",
      projectId: "natick-cny-2026",
      // ...
    };

    const app = initializeApp(firebaseConfig);
    export const db = getFirestore(app);
    ```
3.  **Replace LocalStorage:**
    Modify `services/dataService.ts`. Instead of reading `localStorage.getItem`, use:
    ```typescript
    import { collection, addDoc, query, where, getDocs } from "firebase/firestore";
    // ... implement methods using db instance
    ```

## 5. Validation & Migration Plan

Since you have existing data in `localStorage`, you need a one-time script to push it to Firebase.
*   **Step:** Create a temporary button in `AdminDashboard`.
*   **Logic:** Loop through `getReservations()` (local) -> `await addDoc(collection(db, 'reservations'), res)`.

### 2. Concurrency Test
*   **Test:** Open the Staff Portal on two different devices (Mobile + Laptop).
*   **Action:** Check in "Family A" on Mobile.
*   **Expectation:** The Laptop dashboard should automatically update to show "Family A" as Arrived (Green) without refreshing the page (requires `onSnapshot` listener).

### 3. Offline Capabilities
*   **Config:** Enable `enableIndexedDbPersistence(db)`.
*   **Test:** Turn off Wi-Fi on iPad scanner -> Scan Ticket -> Turn Wi-Fi on.
*   **Expectation:** Check-in syncs to server once connection is restored.
