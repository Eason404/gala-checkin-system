# Security Policy

## Supported Versions

We recommend always using the latest version of the application. Security updates and patches are applied to the active development branch.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of our systems seriously, and we value the security community. The responsible disclosure of security vulnerabilities helps us ensure the security and privacy of our users.

If you believe you have found a security vulnerability in this repository, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them privately using one of the following methods:
1. **GitHub Private Vulnerability Reporting**: If enabled on this repository, please go to the "Security" tab, select "Advisories", and click "Report a vulnerability".
2. **Email**: Alternatively, you can reach out directly to the project maintainers via email.

Please include the following details in your report:
* A descriptive summary of the vulnerability.
* Detailed steps to reproduce the issue, including any necessary configuration or code snippets.
* The potential impact or risk of the vulnerability.
* (Optional) Any suggested mitigations or fixes.

### What to Expect

* We will acknowledge receipt of your vulnerability report as quickly as possible.
* We will investigate the issue and determine the appropriate course of action based on the severity and impact.
* We will keep you updated on the progress of our investigation and any planned fixes.
* Once the issue is resolved, we will publish a security advisory and credit you for the discovery (if you wish).

Thank you for helping us keep this project safe!
